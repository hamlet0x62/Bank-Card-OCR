项目环境简介

项目中，我们完成主要模块使用到的工具和框架如下

  • 神经网络的搭建中使用了Tensorflow 1.10.0 以及numpy 1.14.3
  • 图像数据集的增强和预处理中使用到了 scikit-image 0.13.1

使用到的运行环境是带有 Python3.6.5的Anaconda3-5.1.0

开发环境搭建

建议使用Linux进行环境搭建

依赖安装

在项目根目录下， 运行如下命令

> pip install virtualenv
> virtualenv env 
> source env/bin/activate 
> (env) pip install -r requirements.txt
扩充数据集
  • 先跳转到项目根目录Bank_Card_Ocr， 以下是示例
    > cd Bank_Card_Ocr
  • 使用以下命令扩充银行卡号数据集， 得到TFRecord格式的训练集和验证集
    > python -m prepare.enlarge_card_no_dataset \
--input_image_dir=ORIGIN_IMAGE_DIR \ # 原始图片目录
--eval_output_dir=EVAL_OUTPUT_DIR \ # 验证集输出目录
--train_output_dir=TRAIN_OUTPUT_DIR \ # 训练集输出目录
  • 使用以下命令扩充银行卡数据集， 得到TFRecord格式的训练集和验证集
    > python -m prepare.enlarge_detection_dataset \
--label_path=LABEL_PATH \ # 标注文件目录
--eval_output_dir=EVAL_OUTPUT_DIR \ # 验证集输出目录
--train_output_dir=TRAIN_OUTPUT_DIR \ # 训练集输出目录
卡号检测模型训练环境搭建
  • 在卡号区域检测模型的构建过程中， 使用到了开源项目 tensorflow/models/research下的object_detection模块
    • 需要注意的是， object_detection模块依赖于tensorflow/models/research下的slim模块，slim模块的安装步骤如下
      > cd slim # 下载完slim模块后， 跳转至模块根目录
> python setup.py install
> python setup.py build
  • 安装好相关依赖后，跳转至object_detection目录下， 运行以下命令
    > protoc ./protos/*.proto --python_out=.
  • 使用object_detection做迁移学习， 需要完成以下步骤
    • 确定好特征提取模型（如 VGG-16 等），准备好训练所需数据集
    • 在 detection_model.config 中修改好训练参数和对应的配置项
    • 开始运行 object_detection 的 model_main 进行迁移学习，以下是运行示例
      > python -m object_detection.model_main \
--pipeline_config_path=./detection_model.config \ # 训练配置文件
--model_dir=MODEL_DIR \ # 检查点文件、日志输出目录
--sample_1_of_n_eval_examples=1 \
--num_train_steps=20000 \ # 训练次数
  • 训练完毕后， 使用object_detection下的export_inference_graph导出模型， 以下是运行示例
    > python -m object_detection.export_inference_graph \
--input_type=INPUT_TYPE \ # 输入tensor的类型
--pipeline_config_path=PIPELINE_CONFIG_PATH \ # 训练配置文件路径
--trained_checkpoint_prefix=MODEL_DIR/CHECK_POINT_PREFIX \ # 训练好的检查点文件路径
--output_directory=OUTPUT_DIR # 输出目录
  • 运行完上述命令后， 在OUTPUT_DIR下可找到导出的模型
卡号识别模型开发环境搭建

  • 卡号识别模型项目结构如下

      recognize/
  ├── cnn_model.py # 卷积网络模型
  ├── eval.py # 模型验证模块
  ├── input.py # 模型的输入处理模块
  ├── lstm.py  # LSTM模型
  ├── model_train.py # 模型的训练入口文件
  ├── models # 检查点文件的存放目录
  └── recognize_settings.py # 配置文件

  • 在recognize_settings.py中设置好训练、验证数据集路径后， 运行以下命令训练模型

  • 在Bank_Card_Ocr下找到settings.py配置文件，将其中的DIR2SAVE_CARD_NO_RECORDS以及DIR2SAVE_CARDNO_EVAL_RECORDS修改到对应的训练数据集和验证数据集。然后， 运行以下命令开始训练模型

    > cd Bank_Card_Ocr
> python -m recognize.model_train

  • 训练过程中， 可以使用以下命令开启 tensorboard查看可视化日志

    > tensorboard --logdir ./log-dir/recognize

  • 使用以下命令评估模型

    > python -m recognize.eval

搭建测试环境

搭建测试环境用于测试整合后的模型和图形用户接口(GUI)

图形客户端的部署
  • 我们的图形客户端是一个 Web 应用, 主要使用到了Flask和Vue
  • 启动应用前需要先安装docker
  • 安装docker完毕后， 运行以下命令构建 docker 镜像
    > cd Bank_Card_Ocr/app
> docker build . -t bank_card_ocr_app:latest
... # 等待构建完毕
> docker run -p 5000:5000 bank_card_ocr_app
  • 现在可以登录http://127.0.0.1:5000/访问应用了, 或者你也可以访问我们已经部署好的网页客户端：http://47.102.212.208:5000
运行测试模块

  • 测试的入口位于Bank_Card_Ocr/demo中， demo目录结构如下：

第三方工具、软件信息说明

  • tensorflow/models/research下的object_detection模块是由 Google 开源的一个用于快速搭建对象检测模型的工具
  • tensorflow/model/research下的slim模块中定义了许多常用的神经网络模型
  • numpy 是一个 Python 中常用的高效率科学计算模块
  • scikit-image是一个开源的数字图像处理工具
  • Tensorflow是一个能够快速搭建神经网络模型的人工智能框架
  • TensorBoard是一个为Tensorflow打造的可视化工具， 可以将模型的训练过程可视化
  • docker是一个轻量级的镜像容器管理工具， 可以借助它轻易地实现应用的部署
  • Flask是一个微型的(Micro)Web框架， 我们借助它搭建Web应用的服务器端
  • Vue.js是流行的MVVM前端框架， 我们借助它搭建Web应用的客户端
  • Virtualenv 用于创建一个隔离的Python程序运行环境
